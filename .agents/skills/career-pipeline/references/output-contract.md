# Career Pipeline Output Contract

## `04_기업직무조사.md`

중요 주장마다 공식 출처의 직접 Markdown 링크를 넣는다. 확인된 사실과 `[확인 필요]` 항목을 분리한다.

## `05_문항전략.md`

각 문항에 다음을 넣는다.

- 문항 분류
- 선택 경험
- `02_사실원장.json` 의 정확한 `source_path`
- 핵심 메시지
- 기관·직무 연결
- 부족하거나 확인하지 못한 근거

## `draft.json`

UTF-8 JSON 배열로 작성한다.

```json
[
  {
    "question_index": 1,
    "answer": "공백 포함 제한 글자 수 이내의 답변",
    "evidence_paths": ["경험정리/근거문서.docx"]
  }
]
```

`evidence_paths`는 각 답변에 한 개 이상 넣고, 사실 원장의 `source_path`와 정확히 일치시킨다.

## `08_면접대비팩.md`

다음 제목을 모두 포함한다.

- `1분 자기소개`
- 각 문항의 30·60·90초 답변
- 핵심 주장당 세 개 이상의 `꼬리질문`
- 수치·역할·성과를 따지는 `압박질문`
- 주장을 받치는 정확한 로컬 `근거` 파일

자기소개서에 없는 새로운 주장을 면접 답변에 추가하지 않는다.
