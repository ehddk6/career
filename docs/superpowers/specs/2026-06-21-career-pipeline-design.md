# Career Pipeline Codex 통합 워크플로 설계

## 1. 목적

현재 폴더에 쌓인 자기소개서, 경험정리, 직무기술서, 기업조사, 면접 자료를 하나의 Codex 워크플로로 연결한다. 사용자가 지원 기업·직무와 채용공고를 지정하면 자료 분석, 사실 검증, 최신 웹 조사, 문항별 전략, 자기소개서, 검토 보고서, 면접 대비팩을 한 번에 생성한다.

핵심 원칙은 `결정적 분석 먼저, Codex 합성 나중`이다. 파일 추출, 중복 판별, 수치·기간 충돌, 글자 수, 블라인드 규칙은 로컬 엔진이 판정하고, Codex는 구조화된 사실과 출처를 기반으로 조사·작성한다.

## 2. 사용 형태

저장소 로컬 Codex 스킬 `career-pipeline`을 제공한다.

```text
$career-pipeline
대상: HUG 체험형 인턴 금융·기금(강원)
공고: <URL 또는 로컬 파일>
초안: 26-06-21_주택도시보증공사(HUG) 일반전형_금융·기금(강원).docx
```

자연어 호출도 동일하게 지원한다.

```text
HUG 공고와 현재 자소서를 기준으로 자료조사부터 면접팩까지 실행해줘.
```

스킬은 내부적으로 Python 분석 엔진을 호출하고, 웹 조사와 문장 합성은 Codex 단계로 수행한다. 사용자는 Python 명령을 직접 실행할 필요가 없다.

## 3. 아키텍처

### 3.1 Codex 스킬 계층

- 위치: `.agents/skills/career-pipeline/SKILL.md`
- 역할: 입력 파악, 단계 오케스트레이션, 웹 조사, 산출물 작성, 중단·재개 규칙
- 원칙: 최신 공식 자료 우선, 참고 자소서 문구 복사 금지, 근거 없는 주장 생성 금지

### 3.2 로컬 분석 엔진

- 위치: `career_pipeline/`
- 파일 추출: DOCX, PDF, XLSX, TXT, Markdown
- 인벤토리: 파일 형식, 크기, 수정일, 해시, 문서 유형
- 민감 정보 필터: 비밀번호, 주민번호, 연락처, 성적·자격·병역 증빙 기본 제외
- 사실 후보 추출: 근무 기간, 소속, 역할, 행동, 성과, 수치, 근거 파일
- 충돌 탐지: 동일 경험의 기간·수치·성과·역할이 다른 경우
- 규칙 검사: 글자 수, 타기관명, 블라인드, 공백 문항, 반복 문구

### 3.3 문서 생성기

- Markdown을 기준 산출물로 삼는다.
- 제출용 자기소개서는 Markdown과 DOCX를 함께 생성한다.
- DOCX는 문항, 제한 글자 수, 답변, 출처·검토 메모를 분리하여 제출 본문에 검토 메모가 섞이지 않게 한다.

## 4. 데이터 흐름

1. 채용공고, 초안, 지원 기업·직무, 글자 수 조건을 입력받는다.
2. 작업공간을 스캔하여 파일을 `사용`, `제외`, `중복`, `확인 필요`로 분류한다.
3. 문서 본문에서 사실 후보와 문항 제약을 추출한다.
4. 사실 후보를 경험별로 군집하고 서로 다른 수치·기간·역할을 충돌로 기록한다.
5. 충돌이 하나라도 있으면 작성 단계를 멈추고 `03_충돌검사.md`에 확인 질문을 생성한다.
6. 충돌이 해결되면 기업 공식 홈페이지, 채용공고, 알리오·공시, 업무 규정 등을 최신 웹 자료로 확인한다.
7. 문항을 지원동기, 직무역량, 문제해결, 협업·신뢰, 윤리, 포부 등으로 분류하고 검증된 경험을 배치한다.
8. 자기소개서를 작성하고 글자 수·블라인드·타기관명·주장 근거를 재검사한다.
9. 최종 자기소개서의 각 주장을 기준으로 기본 질문, 꼬리질문, 압박질문, 방어 근거를 생성한다.

## 5. 실행 산출물

각 실행은 `career_runs/<기업-직무-YYYYMMDD-HHMMSS>/`에 독립적으로 저장한다.

| 파일 | 내용 |
|---|---|
| `01_자료목록.md` | 사용·제외·중복·추출 실패 파일 |
| `02_사실원장.json` | 경험, 기간, 역할, 행동, 성과, 수치, 근거 |
| `03_충돌검사.md` | 충돌 내용, 원문 출처, 사용자 확인 질문 |
| `04_기업직무조사.md` | 최신 공식 자료 요약과 링크 |
| `05_문항전략.md` | 문항 분류, 경험 배치, 핵심 메시지, 부족 근거 |
| `06_자기소개서.md` | 문항별 최종 답변 |
| `06_자기소개서.docx` | 제출·복사 용이한 Word 본문 |
| `07_자기소개서_검토보고서.md` | 글자 수, 근거, 위험 주장, 블라인드, 타기관명 |
| `08_면접대비팩.md` | 1분 자기소개, 예상·꼬리·압박질문, STAR 답변, 방어 근거 |
| `run.json` | 입력, 실행 단계, 출처, 오류, 산출물 해시 |

## 6. 사실 충돌 처리

초안 작성 중단이 기본 정책이다. 예를 들어 동일한 숙박비 검증 경험의 절감액이 `1천만 원`, `2천만 원`, `4천만 원`, `5천만 원`, `1억 원`, `40%`로 다르면 어떤 값도 자동 채택하지 않는다.

사용자가 확정한 값은 해당 실행의 `fact_overrides.yaml`에 기록하고, 원문은 바꾸지 않는다. 재개 시 확정 값과 근거를 사실 원장에 함께 기록한다.

## 7. 웹 조사 정책

- 매 실행마다 최신 웹 자료를 확인한다.
- 출처 우선순위는 채용공고 원문, 기업·기관 공식 사이트, 정부·알리오·공시, 업무 규정, 공신력 있는 보도 자료 순이다.
- 핵심 주장 마다 직접 링크를 남긴다.
- 공식 자료로 확인하지 못한 정보는 `[확인 필요]` 또는 `[NEEDS_VERIFICATION]`으로 표시하고 제출 본문에서 사용하지 않는다.
- 로컬 문서의 개인정보를 검색어·URL·외부 폼에 전송하지 않는다.

## 8. 자기소개서 작성 규칙

- 사실 원장에서 `confirmed`로 표시된 정보만 사용한다.
- 지원 기업의 문제와 사용자 경험을 연결하되 인과관계를 과장하지 않는다.
- 참고 자소서는 질문 분류·구조 파악에만 사용하고 문구를 복사하지 않는다.
- 매 문항은 제한 글자 수를 초과하지 않는다.
- 안전 범위는 제한 글자 수의 92~98%로 설정한다.
- 검증이 어려운 정확한 수치보다 면접에서 설명 가능한 표현을 우선한다.
- 성명, 출신학교, 출신지역, 생년월일, 가족관계 등 블라인드 위반 후보를 제출 전에 검사한다.

## 9. 면접 대비팩 규칙

면접팩은 최종 자기소개서에서 새로운 주장을 추가하지 않는다. 각 문항에 대해 다음을 생성한다.

- 30·60·90초 답변
- 기본 질문과 꼬리질문
- 수치·역할·성과를 따지는 압박질문
- 주장을 받치는 근거 파일과 출처
- 허위·과장 오해를 막기 위한 안전한 답변 선
- 공통 질문: 1분 자기소개, 지원동기, 강점·약점, 갈등, 윤리, 직무 이해, 입사 후 포부

## 10. 오류 처리·재개

- 사실 충돌: 산출물 1~3까지 보존하고 작성을 중단한다.
- 추출 실패: 해당 파일만 제외하고 파일명·오류를 `01_자료목록.md`에 기록한다.
- 공식 출처 부족: 해당 정보를 제출 본문에서 제외하고 조사 보고서에 확인 필요로 남긴다.
- DOCX 생성 실패: Markdown 산출물을 보존하고 실패 원인과 재실행 명령을 기록한다.
- 재개: 기존 `run.json`과 `fact_overrides.yaml`을 읽어 완료된 단계는 재사용하고, 입력 해시가 바뀌 단계만 다시 실행한다.

## 11. 보안·개인정보

다음 파일과 폴더는 기본 제외한다.

- `Chrome 비밀번호.csv`
- `학교성적/`
- `자격증/`
- `경력증명서/`
- 병적·주민번호·전화번호·이메일·주소가 포함된 증빙

자격·학력·경력 정보가 지원 조건 검증에 필요한 경우에만 사용자가 특정 파일을 명시적으로 포함하도록 한다. 추출된 민감 값은 로그·웹 검색·산출물에 기록하지 않는다.

## 13. 검증 계획

### 13.1 자동 테스트

- DOCX, PDF, XLSX, TXT 샘플에서 본문·문항·글자 수 추출
- 중복 파일 SHA-256 탐지
- 민감 파일 기본 제외
- 동일 경험의 서로 다른 절감액·기간·실적 충돌 탐지
- 충돌 존재 시 자기소개서 생성 차단
- 글자 수, 타기관명, 블라인드, 빈 답변 검사
- Markdown과 DOCX의 문항 수·본문 내용 일치
- `run.json`을 이용한 중단 후 재개

### 13.2 실제 자료 통합 테스트

현재 HUG 지원서를 대상으로 전체 워크플로를 실행한다.

- 현재 4개 빈 문항 인식
- 과거 HUG 초안과 경험 자료 연결
- 숙박비 절감액 충돌 검출로 작성 단계 중단
- 사용자 확정 값 반영 후 재개
- 최신 HUG 공식 자료 출처 포함
- 4개 문항 각 600자 이내 초안
- DOCX·검토 보고서·면접팩 생성

## 14. 완료 기준

다음 조건이 모두 충족되어야 완료로 간주한다.

1. Codex에서 `career-pipeline`을 명시적 스킬 또는 자연어로 호출할 수 있다.
2. 지원 공고와 초안을 주면 현재 폴더 자료를 자동 분류한다.
3. 수치·기간·역할 충돌을 검출하고 해결 전에는 초안을 작성하지 않는다.
4. 매 실행에서 최신 공식 웹 자료를 확인하고 출처를 남긴다.
5. 문항 전략, 자기소개서 Markdown·DOCX, 검토 보고서, 면접 대비팩을 생성한다.
6. 민감 파일은 기본 제외되고 개인정보가 웹으로 전송되지 않는다.
7. 자동 테스트와 HUG 실제 자료 통합 테스트가 통과한다.
