# GPT-5.6 Sol 외부 프롬프트와 취업 프로젝트 비교

## 결론

외부 프롬프트는 실행 단계를 명시하는 설계 문서로서 강점이 있고, 취업 프로젝트는 실제 제출 안전성을 보장하는 결정론적 시스템으로서 강점이 있다. 외부 프롬프트의 강점을 다음 네 가지로 흡수했다.

- `data_package_id`·버전·동결 SHA-256으로 단계 간 자료 혼용 방지
- `EXPERIENCE_DIVERSITY` 후보 전략과 대체 경험 비교
- 결정론적 HARD FAIL과 의미적 `REVIEW_REQUIRED` 분리
- 심사위원 2명 이상이 인정한 이식 요소와 최종 X/Y 전체 버전 비교

## 비교표

| 항목 | 외부 프롬프트 | 취업 프로젝트 | 통합 결과 |
|---|---|---|---|
| 원자료 분리 | `input/frozen` 디렉터리 | 원장·공고·research·manifest 분리 | 프로젝트의 manifest·SHA 구조 유지 |
| 사실 통제 | 사실·해석 표 분리 | claim ID·검증 객체·기여 범위 | 프로젝트 claim 검증을 기준으로 통합 |
| 경험 배치 | 대체 경험·재사용 점수 | 매칭 점수·재사용 패널티 | 4점 미만 차이면 미사용 대체 경험 우선 |
| 후보 전략 | FACT / QUESTION / DIVERSITY / JOB | FACT / QUESTION / VOICE / JOB | DIVERSITY를 코드와 심사 프롬프트에 반영 |
| 심사 오류 | 결정론적·의미적 오류 구분 | 기존 boolean HARD FAIL | 의미 오류는 재검토, 확정 오류만 탈락 |
| 심사 근거 | 이식 요소·fact ID | 점수·weakness code | 이식 요소를 2명 이상 동의 시 합성 허용 |
| 최종 선택 | X/Y 전체 버전 선택 | X/Y 선택 | 혼합본 금지와 위험 감사 추가 |
| 글자 수 | 미확인 기준을 보류 | 공식 공고 불일치 시 차단 | 공고 불일치 차단을 유지 |
| 제출 준비 | 파일 산출 중심 | portfolio 6개 게이트 | 포트폴리오 `ready` 게이트 유지 |

## 의도적으로 흡수하지 않은 부분

외부 프롬프트의 `UNVERIFIED` count mode에서 길이만으로 후보를 탈락시키지 않는 규칙은 그대로 사용하지 않았다. 이 프로젝트는 실제 공고 원문과 문항·최소/최대 글자 수가 일치하지 않으면 `blocked_posting`으로 중단해야 하므로, 제출 안전성 측면에서 더 엄격한 현재 정책이 적합하다.

또한 외부 프롬프트의 점수 자체를 합격 확률처럼 사용하지 않았다. 점수는 후보 비교 보조 자료로만 남기고, 최종 상태는 사실·공고·공식 조사·manifest·면접팩·지원 자격 게이트로 결정한다.
